<analysis>**original_problem_statement:**
The user wants to create a Telegram Web App for managing and monitoring the sanitary conditions of rooms in a dormitory.

PRODUCT REQUIREMENTS:
- The application should cover floors 2 through 9. Each floor has 15 blocks.
- Each block contains a Small room, a Big room, and a Common space.
- Users should be able to click on a block to see the residents of each room. Each block has a designated block monitor.
- A rating system from 1 to 5 should be available for the Small room, Big room, and Common space.
- Rooms with a rating of 1-3 should be highlighted in red to indicate they are problem rooms.
- The application must support filtering checks by date and searching for blocks by number or by resident name.
- All data, including resident information and ratings, must be stored on a server.
- There should be a separate login system for floor monitors (one per floor) who are responsible for submitting the ratings.
- A comprehensive admin panel is required for a superuser who can perform all actions, including adding/removing residents, editing ratings, and more.
- The admin panel must include a feature to track logins of floor monitors to prevent account sharing.
- The admin needs the ability to export reports of all ratings from each floor in PDF format.
- An additional feature requested is a schedule for buses and trolleybuses at the nearest stop (Дом правосудия in Minsk), with a humorous message like You better move! if a bus is arriving soon.
- The user also provided a complete list of residents to be populated into the system.
